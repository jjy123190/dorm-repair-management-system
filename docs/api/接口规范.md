# 接口规范

> 说明：本文件仅作历史归档，当前项目开发不再以本文件为准。

## 1. 基础规则

| 项 | 规则 |
| --- | --- |
| 基础前缀 | `/api/v1` |
| 路径风格 | 复数资源名 + `kebab-case` |
| 数据格式 | `application/json` |
| 时间格式 | `yyyy-MM-dd HH:mm:ss` 或 ISO-8601 |
| 分页参数 | `page` 从 `1` 开始，`size` 默认 `20` |
| 列表响应 | 统一使用 `PageResponse` |

## 2. HTTP 方法约定

| 操作 | 方法 | 示例 |
| --- | --- | --- |
| 新增 | `POST` | `POST /api/v1/repair-requests` |
| 查询详情 | `GET` | `GET /api/v1/work-orders/1` |
| 分页查询 | `GET` | `GET /api/v1/dorm-rooms?page=1&size=20` |
| 更新 | `PUT` | `PUT /api/v1/dorm-rooms/1` |
| 更新状态 | `PATCH` | `PATCH /api/v1/work-orders/1/status` |
| 删除 | `DELETE` | `DELETE /api/v1/dorm-rooms/1` |

## 3. 响应包结构

### 普通响应

```json
{
  "success": true,
  "message": "OK",
  "data": {}
}
```

### 分页响应

```json
{
  "success": true,
  "message": "OK",
  "data": {
    "list": [],
    "page": 1,
    "size": 20,
    "total": 0,
    "totalPages": 0
  }
}
```

## 4. 字段命名规则

| 类型 | 规则 | 示例 |
| --- | --- | --- |
| JSON 字段 | `camelCase` | `repairRequestId` |
| URL 路径 | `kebab-case` | `repair-requests` |
| Java DTO | 写入 `*Command`，读取 `*Response` | `CreateRepairRequestCommand` |
| 数据库表 | `snake_case` | `repair_requests` |
| 数据库字段 | `snake_case` | `created_at` |

## 5. 状态与枚举规则

- 前后端统一传枚举英文码
- 页面显示中文由前端映射
- 后端文档中明确列出可用枚举值

### 故障类型

- `ELECTRICITY`
- `WATER_PIPE`
- `DOOR_WINDOW`
- `NETWORK`
- `FURNITURE`
- `PUBLIC_AREA`
- `OTHER`

### 报修状态

- `SUBMITTED`
- `ASSIGNED`
- `IN_PROGRESS`
- `COMPLETED`
- `REJECTED`
- `CANCELLED`

### 工单状态

- `ASSIGNED`
- `ACCEPTED`
- `IN_PROGRESS`
- `WAITING_PARTS`
- `COMPLETED`
- `REJECTED`
- `CANCELLED`

### 工单优先级

- `LOW`
- `NORMAL`
- `HIGH`
- `URGENT`

## 6. 变更规则

- 改 URL，必须同步改接口清单
- 改请求字段，必须同步改接口清单和数据库设计
- 改分页结构，必须先和前端同事对齐
