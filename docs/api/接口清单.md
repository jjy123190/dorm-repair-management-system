# 接口清单

> 以下是当前约定的真实接口。前端按这里对接，后端按这里实现，文档同事按这里写说明书。

## 1. 宿舍基础信息

| 接口名 | 方法 | 路径 | 用途 |
| --- | --- | --- | --- |
| 创建宿舍信息 | `POST` | `/api/v1/dorm-rooms` | 新增宿舍房间基础信息 |
| 修改宿舍信息 | `PUT` | `/api/v1/dorm-rooms/{id}` | 修改宿舍房间基础信息 |
| 查询宿舍详情 | `GET` | `/api/v1/dorm-rooms/{id}` | 查看单条宿舍信息 |
| 分页查询宿舍 | `GET` | `/api/v1/dorm-rooms` | 查看宿舍列表 |
| 删除宿舍信息 | `DELETE` | `/api/v1/dorm-rooms/{id}` | 删除一条宿舍信息 |

### 创建/修改宿舍请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| campusName | string | 是 | 校区名称 |
| buildingNo | string | 是 | 楼栋号 |
| roomNo | string | 是 | 房间号 |
| floorNo | number | 是 | 楼层 |
| bedCount | number | 是 | 床位数 |

## 2. 报修申请

| 接口名 | 方法 | 路径 | 用途 |
| --- | --- | --- | --- |
| 提交报修申请 | `POST` | `/api/v1/repair-requests` | 学生提交报修 |
| 查询报修详情 | `GET` | `/api/v1/repair-requests/{id}` | 查看报修详情 |
| 分页查询报修单 | `GET` | `/api/v1/repair-requests` | 学生或管理员查看报修列表 |

### 提交报修请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| studentName | string | 是 | 学生姓名 |
| buildingNo | string | 是 | 楼栋号 |
| roomNo | string | 是 | 房间号 |
| faultCategory | string | 是 | 故障类型枚举 |
| description | string | 是 | 故障描述 |
| contactPhone | string | 是 | 联系电话 |
| imageUrls | string[] | 否 | 图片 URL 数组 |

### 分页查询参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| status | string | 否 | 报修状态 |
| page | number | 否 | 页码，默认 1 |
| size | number | 否 | 每页条数，默认 20 |

## 3. 工单管理

| 接口名 | 方法 | 路径 | 用途 |
| --- | --- | --- | --- |
| 创建并派发工单 | `POST` | `/api/v1/work-orders` | 管理员创建工单并派发 |
| 查询工单详情 | `GET` | `/api/v1/work-orders/{id}` | 查看单条工单 |
| 分页查询工单 | `GET` | `/api/v1/work-orders` | 查看工单列表 |
| 更新工单状态 | `PATCH` | `/api/v1/work-orders/{id}/status` | 维修人员更新工单状态 |

### 创建工单请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| repairRequestId | number | 是 | 报修单 ID |
| adminId | number | 是 | 管理员 ID |
| workerId | number | 是 | 维修人员 ID |
| priority | string | 是 | 工单优先级 |
| assignmentNote | string | 否 | 派单备注 |

### 查询工单参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| status | string | 否 | 工单状态 |
| workerId | number | 否 | 维修人员 ID |
| page | number | 否 | 页码，默认 1 |
| size | number | 否 | 每页条数，默认 20 |

### 更新工单状态请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| status | string | 是 | 工单状态 |
| operatorId | number | 是 | 当前操作人 ID |
| note | string | 否 | 处理说明 |

## 4. 评价反馈

| 接口名 | 方法 | 路径 | 用途 |
| --- | --- | --- | --- |
| 提交评价 | `POST` | `/api/v1/repair-feedbacks` | 学生对已完成报修提交评分 |

### 提交评价请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| repairRequestId | number | 是 | 报修单 ID |
| rating | number | 是 | 评分，1-5 |
| feedbackComment | string | 否 | 评价内容 |
| anonymous | boolean | 否 | 是否匿名 |

## 5. 报表

| 接口名 | 方法 | 路径 | 用途 |
| --- | --- | --- | --- |
| 查询报修月度汇总 | `GET` | `/api/v1/reports/repair-requests/monthly-summary` | 管理员查看月度统计 |

### 月度汇总参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| month | string | 否 | 月份，格式 `yyyy-MM`，为空则取当前月 |

## 6. 当前前端必接接口

- `/api/v1/repair-requests`
- `/api/v1/work-orders`
- `/api/v1/work-orders/{id}/status`
- `/api/v1/repair-feedbacks`
- `/api/v1/reports/repair-requests/monthly-summary`
- `/api/v1/dorm-rooms`
