# 桌面端模块与服务约定

## 1. 模块清单

| 模块 | 对应类 | 当前职责 |
| --- | --- | --- |
| 首页概览 | `DashboardModule` | 展示核心统计和最近报修 |
| 学生报修 | `StudentRepairModule` | 预留报修表单和历史记录区 |
| 管理员派单 | `AdminDispatchModule` | 展示待派单列表和派单流程 |
| 维修处理 | `WorkerProcessingModule` | 展示活动工单和状态更新流程 |
| 月度统计 | `StatisticsModule` | 展示月度汇总表 |

## 2. 服务入口

### `DashboardService`

- `loadOverview()`
- `listRecentRepairRequests(int limit)`

### `RepairRequestService`

- `create(CreateRepairRequestCommand command)`
- `listLatestSubmittedRequests(int limit)`
- `listPendingAssignmentRequests(int limit)`
- `submitFeedback(SubmitRepairFeedbackCommand command)`

### `WorkOrderService`

- `assign(AssignWorkOrderCommand command)`
- `updateStatus(UpdateWorkOrderStatusCommand command)`
- `listActiveWorkOrders(int limit)`

### `StatisticsService`

- `listMonthlySummary(int recentMonths)`

## 3. Command 约定

- 界面层不要直接 new 实体写库，统一先组装 `command`
- `command` 只放用户输入和上下文，不放数据库自增 ID 生成逻辑
- 业务编号统一由 `BusinessNumberGenerator` 生成

## 4. Mapper 约定

- 一个业务表或一个统计主题对应一个 Mapper
- SQL 统一写在 `src/main/resources/mapper/*.xml`
- `mapper` 接口和 XML `namespace` 必须一一对应
- 尽量让 SQL 查询结果直接映射到 `domain.view`，不要让 UI 自己拼字段

## 5. 状态流转约定

- 报修单状态：`SUBMITTED -> ASSIGNED -> IN_PROGRESS -> COMPLETED`
- 异常分支：`REJECTED`、`CANCELLED`
- 工单状态：`ASSIGNED -> ACCEPTED -> IN_PROGRESS -> WAITING_PARTS -> COMPLETED`
- 工单状态更新后，要同步更新报修单状态
- 每次工单状态变化，都要插入 `work_order_records`

## 6. UI 编码约定

- 模块类只负责布局、表格、按钮事件绑定
- 不要在 JavaFX 页面里直接打开 `SqlSession`
- 表格展示对象统一使用 `domain.view`
- 长流程逻辑统一下沉到 `service.impl`
