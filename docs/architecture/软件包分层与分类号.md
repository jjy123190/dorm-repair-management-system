## 命名规则

### Java 类名

- 实体：名词单数，例如 `RepairRequest`
- 命令对象：动作 + 业务名 + `Command`
- 展示对象：业务名 + `View` / `Summary`
- Mapper：业务名 + `Mapper`
- Service 接口：业务名 + `Service`
- Service 实现：接口名 + `Impl`
- 模块页：业务名 + `Module`

### 包内职责

- `domain.entity` 只放实体，不放流程逻辑
- `mapper` 只写 SQL，不写业务校验
- `service.impl` 负责事务、状态流转、异常判断
- `ui.module` 只负责布局和用户交互

## 后续扩展规则

如果后续代码继续增长，可以再按业务域继续细拆：

- `repair`
- `workorder`
- `feedback`
- `report`
- `basicdata`

在那之前，优先保持当前分层稳定，不做无收益的大规模迁移。
