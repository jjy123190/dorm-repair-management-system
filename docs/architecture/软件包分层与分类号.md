## 命名规则

### Java 类名

- 实体：名词单数，例如 `RepairRequest`
- 枚举：业务名 + `Status` / `Priority` / `Role`
- Repository：实体名 + `Repository`
- Service 接口：实体或业务名 + `Service`
- Service 实现：接口名 + `Impl`
- Controller：资源名 + `Controller`
- 写接口入参：`*Command`
- 读接口出参：`*Response`

### 包内职责

- `domain.entity` 只放实体，不放业务逻辑
- `repository` 只做数据库访问，不写流程判断
- `service.impl` 负责流程、事务、状态流转
- `web.controller` 只收发请求，不堆业务逻辑

## 后续扩展规则

如果后续代码继续增长到 80+ Java 文件，下一步再拆成按业务模块分包：

- `dorm`
- `repair`
- `workorder`
- `feedback`
- `report`

在那之前，保持当前分层结构稳定，不做大规模包迁移。
