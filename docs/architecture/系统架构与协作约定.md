# 系统架构与协作约定

## 1. 当前目标

当前仓库不是 Web 后端脚手架，而是给小组成员继续开发的桌面应用基础架构。

主线目标：

- 用 `JavaFX` 承载学生、管理员、维修员、统计四类操作界面
- 用 `MyBatis` 直写 SQL，突出数据库课设能力
- 用 `MySQL` 落地工单、报修、评价、统计数据

## 4. 各层职责

- `common`：手写应用上下文、MyBatis 事务模板、业务编号生成
- `config`：读取 `application.yml`，装配 HikariCP 和 MyBatis
- `domain.command`：界面层传给 service 的命令对象
- `domain.entity`：与数据库表对应的实体对象
- `domain.enums`：角色、状态、优先级、故障类型
- `domain.view`：工作台表格和统计页使用的只读展示对象
- `mapper`：只负责 SQL 和结果映射
- `service`：定义业务能力边界
- `service.impl`：负责事务、流程和状态同步
- `ui`：JavaFX 主工作台
- `ui.module`：首页、学生、管理员、维修员、统计等模块页面

## 5. 业务模块边界

- 首页概览模块：总数、待审核、进行中、本月完成、最近报修
- 学生报修模块：提交报修、查看历史、后续接图片上传
- 管理员派单模块：待派单列表、优先级、派单说明、催办
- 维修处理模块：接单、处理中、待配件、完成
- 月度统计模块：月度汇总、完成率、后续扩展图表和导出

## 7. 演进规则

- 新增业务先定表和字段，再定 command，再写 service 和 mapper，最后补 UI
- 不要跳过 `service` 直接在 UI 里写 SQL
- 不要在 `mapper` 里堆流程判断
- 枚举状态一旦进库，不要随意改英文码
- `docs/api/` 目前只作为旧 REST 存档，不再作为主线实现依据
